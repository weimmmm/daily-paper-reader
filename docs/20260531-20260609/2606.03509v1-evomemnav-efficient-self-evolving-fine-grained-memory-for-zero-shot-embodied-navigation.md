## 摘要
构建记忆对于零样本具身导航中的长时程规划至关重要。以检测器为中心的场景图通常将观测压缩为稀疏节点，丢弃细粒度视觉证据并积累噪声，而基于3D重建的方法在计算上仍然难以承受。我们提出EvoMemNav，一种用于零样本具身导航的高效、自进化、细粒度记忆框架。EvoMemNav构建了一个视觉-语义记忆图（VSMGraph），将原始视图作为一等记忆，并通过轻量级语义线索和拓扑关系组织成房间-视图-对象层次结构，保留细粒度细节以用于消歧和停止验证。为了扩展到不断增长的记忆，我们引入了一种有预算的粗到细策略：粗阶段将搜索空间压缩到有希望的区域，细阶段仅调用VLM进行有针对性的验证和决策。除了静态记忆，EvoMemNav在每个子任务后执行反思驱动的写回，更新图附加的先验知识，这些先验编码了累积的环境知识，以优化未来的决策而无需重新训练。在GOAT-Bench和HM3D上的实验涵盖了对象、文本描述和图像目标模态，在SR/SPL上取得了一致的提升，具有更好的多实例消歧、更少的过早停止和更强的零样本泛化能力。

## Abstract
Building memory is essential for long-horizon planning in zero-shot embodied navigation. Detector-centric scene graphs often compress observations into sparse nodes, discarding fine-grained visual evidence and accumulating noise, while 3D reconstruction-based methods remain computationally prohibitive. We present EvoMemNav, an efficient, self-evolving, fine-grained memory framework for zero-shot embodied navigation. EvoMemNav constructs a Visual-Semantic Memory Graph (VSMGraph) that keeps raw views as first-class memory and organizes them with lightweight semantic cues and topological relations into a room-view-object hierarchy, preserving fine-grained details for disambiguation and Stop verification. To scale to growing memory, we introduce a budgeted coarse-to-fine policy: a coarse stage compresses the search space into promising regions, and a fine stage invokes a VLM only for targeted verification and decision. Beyond static memories, EvoMemNav performs reflection-driven write-back after each subtask, updating graph-attached priors that encode accumulated environmental knowledge to refine future decisions without retraining. Experiments on GOAT-Bench and HM3D across object, text-description, and image-goal modalities show consistent gains in SR/SPL, with better multi-instance disambiguation, fewer premature stops, and stronger zero-shot generalization.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：零样本具身导航（Zero-Shot Embodied Navigation）中，智能体需要在未见过的环境中依据多模态目标（物体类别、文本描述、参考图像）进行长时程导航。现有记忆方法存在两大不足：
  - **检测器驱动的场景图**：将观测压缩为稀疏节点（如物体、房间），丢弃细粒度视觉证据（纹理、空间布局），且对检测噪声敏感，无法让强大的VLM直接对多视图图像进行推理。
  - **3D重建方法**：计算开销大，与VLM兼容性差。
- **现存图像基记忆的局限**：尽管存储原始视图并连接拓扑，但缺乏结构化组织（房间、边界、可达性），易导致同类物体误判和过早停止；且记忆不具备进化能力，无法从成功/失败中学习。
- **研究含义**：本文旨在构建一种高效、自进化的细粒度记忆框架，保留原始视觉证据，组织为层次化结构，并通过轻量级先验实现持续改进，无需重新训练，从而提升零样本导航的泛化能力和鲁棒性。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- 构建**视觉-语义记忆图（VSMGraph）**：将原始视图作为一等记忆，组织为“房间-视图-对象”层次结构，保留细粒度信息。
- 采用**有预算的粗到细导航策略**：粗阶段压缩候选空间至小集合，细阶段仅对该短列表调用VLM进行目标选择和结构化停止验证。
- 引入**反思驱动的持续记忆适应（RDCMA）**：每个子任务后根据结果写回图附加的先验（短期、长期），指导未来决策，无需重新训练。

### 关键技术细节
1. **VSMGraph构造**：
   - **节点类型**：房间节点（索引视图和对象）、视图节点（分为锚定视图——存储对象丰富区域的原始证据；边界视图——表示可达的探索边界）、对象节点（轻量级实例假设）。
   - **边类型**：可导航边（连续的免碰撞视点间）、可见性边（视图到其观察到的对象）。
   - **属性**：每个视图节点附加CLIP导出的离散房间类型标签（用于粗阶段分桶）和对象命中标签（源于轻量级实例缓存，仅用于检索，最终决策依赖多视图图像）。
   - **拓扑骨架**：占据网格提供度量支持，VSMGraph支持高层推理。
2. **粗到细导航策略**：
   - **粗阶段（Explore）**：基于房间标签分桶和Top-K压缩锚定视图（$K_A$）和边界视图（$K_F$），生成紧凑候选集。若锚定集为空，直接路由到边界视图。
   - **细阶段**：
     - **Search**：在紧凑集上查询VLM，输出所选节点及置信度（确定、不确定、未知）。若低置信度，退回探索路由到边界视图。
     - **Verify**：到达所选锚定视图后，调用结构化停止验证（多视图图像），返回STOP或RESELECT。RESELECT触发锚定视图冷却并重新进入粗阶段。
   - **Recover**：死锁时暂时强制边界探索。
3. **RDCMA**：
   - **Episode-STM**：短期记忆，当前episode内抑制和避免循环，episode结束时重置。
   - **Scene-LTM**：长期记忆，跨子任务/场景持续保留，带衰减、Top-K上限和冲突抑制。
   - **使用方式**：探索阶段作为分桶内的保守打破平局先验（偏向高历史效用和低死胡同风险的边界/锚定视图）；停止校准阶段在验证提示中注入停止可靠性提示。
4. **算法流程**：
   - 智能体接收多模态目标和RGB-D流。
   - 在线更新VSMGraph（添加视图节点、更新占据网格、生成边界视图）。
   - 重复：粗阶段→若需VLM则细阶段（Search+Verify）→否则直接导航→到达后Verify→若STOP则结束，若RESELECT则继续。
