## 摘要
实例目标导航（IGN）要求具身智能体根据不明确的自然语言描述，在干扰物中找到特定对象实例。这种歧义通常无法仅通过感知和语言解决，因此与 oracle 的交互成为消除歧义的自然机制。先前的交互方法允许 oracle 查询，但将轻量级澄清和路线级指导等同对待，使得智能体通过重复的高信息量问题来提高成功率，而非高效地解决底层歧义。我们将交互式 IGN 重新定义为成本敏感的不确定性降低问题，其中智能体应提出其答案能相对于惩罚最大程度降低导航不确定性的问题。为此，我们对现有导航语料库进行信息增益分析，以识别哪些线索能降低导航不确定性，从而得到一组紧凑的问题类型和数据导出的权重。然而，现有的交互式导航基准没有建模不同问题类型的成本，也没有评估智能体使用交互的效率，因此不适合研究成本敏感的交互。基于这一分类，我们构建了一个用于诊断交互行为和效率的基准，以及一个加权成功率指标，该指标根据每个查询的导出成本对其进行惩罚。此外，我们提出了一种零样本 MLLM 导航器，该导航器仅在预期不确定性降低证明交互成本合理时，才在每个决策步骤有选择地进行查询。

## Abstract
Instance Goal Navigation (IGN) requires an embodied agent to find a specific object instance among distractors from an under-specified natural-language description. Such ambiguity often cannot be resolved from perception and language alone, making interaction with an oracle a natural mechanism for disambiguation. Prior interactive methods allow oracle queries but treat lightweight clarification and route-level guidance alike, letting agents boost success rate through repeated high-information questions rather than by resolving the underlying ambiguity efficiently. We recast interactive IGN as a cost-sensitive uncertainty-reduction problem, where the agent should ask the question whose answer provides the largest reduction in navigation uncertainty relative to its penalty. To this end, we apply an information-gain analysis on existing navigation corpora to identify which cues reduce navigation uncertainty, yielding a compact set of question types and data-derived weights. However, existing interactive navigation benchmarks do not model the cost of different question types or evaluate how efficiently agents use interaction, making them unsuitable for studying cost-sensitive interaction. Based on this taxonomy, we construct a benchmark for diagnosing interaction behavior and efficiency, together with a Weighted Success Rate metric that penalizes each query by its derived cost. We further propose a zero-shot MLLM navigator that selectively queries at each decision step only when the expected uncertainty reduction justifies the interaction cost.

## 1. 核心问题与整体含义（研究动机和背景）

- **任务定义**：实例目标导航（Instance Goal Navigation, IGN）要求具身智能体根据模糊的自然语言描述（如“把杯子拿给我”），在存在同类别干扰物的场景中找到特定目标实例。
- **挑战**：仅靠感知和语言无法消除歧义（如房间边界模糊、多个同类别物体、路线冲突），因此与知情 oracle 的交互成为一种自然的消歧机制。
- **现有方法不足**：先前交互式导航方法虽然允许智能体查询 oracle，但将轻量级澄清（如确认属性）与高信息量的路线级指导等同对待，导致智能体通过反复提出高信息量问题来堆叠成功率，而非高效解决底层不确定性；同时缺乏对查询成本的建模和评估。
- **本文目标**：将交互式 IGN 重新定义为**成本敏感的不确定性降低问题**，让智能体在交互时权衡信息增益与查询成本，实现高效消歧。

## 2. 方法论

### 核心思想
智能体应提出其答案能最大程度降低导航不确定性、且相对于查询惩罚代价合理的那些问题。通过分析人类导航语料库，导出问题类型的效用权重，并构建成本感知的交互协议。

### 关键技术细节

#### a) 问题效用建模（Question Utility Modeling）
- 基于**期望信息增益（Expected Information Gain, EIG）** 定义问题价值：  
  `EIG(q) = H[pt(S)] - E_a~p(a|q)[H[pt(S|q,a)]]`  
  其中 S 为隐藏目标状态。
- 直接计算不可行，故使用现有导航语料库（R2R, REVERIE, RxR, CVDN, SOON）作为经验代理，用 Qwen3.5-4B 标注每条记录中的线索（cue）及重要性得分，导出**17种线索类别**，再聚合为4种问题类型。
- 每种线索 c 的复合效用分数：  
  `Uc = λ1·s̄_c + λ2·h_c + λ3·r_c`  
  其中 s̄_c 为平均重要性，h_c 为高增益比例，r_c 为排名第一的比例。  
- 通过提及加权平均聚合到问题类型，再线性缩放得到锁定的每类成本权重（Penalty wt）：
  - Type 1（外观）：0.182
  - Type 2（区域/房间）：0.162
  - Type 3（方向/路线）：0.240
  - Type 4（确认）：0.103

#### b) 基准构建（Benchmark Construction）
- 在 Isaac Sim 中构建实例级 IGN 基准，包含：场景元数据、对象实例与房间图、可控同类别干扰物、四种类型 oracle 答案的生成。
- 每集给出粗略目标（如“找到一台笔记本电脑”），智能体需通过交互识别正确实例，无人类轨迹。
- 难度评分基于：干扰物数量、同房间干扰物、上下文歧义、初始路径距离，分为 easy/medium/hard。
- 从 262 场景、70 目标类别、22,905 集中采样 500 集（easy:medium:hard ≈ 30:40:30）作为评估子集。

#### c) 加权成功率指标（Weighted Success Rate）
- 定义：`Weighted SR(e) = 1{d_e ≤ τ} · exp(- Σ w_type(q_{e,i}))`  
  其中 d_e 为最终到目标距离，τ 为阈值，w_type 为问题类型权重。
- 成功且无问题则退化为普通 SR，失败为 0。重复提问多次计费。

#### d) TANDEM 导航器（Two-stage Navigation with Disentangled Planning and Metric Grounding）
- **架构**：零样本 MLLM 导航器，分为 Planner（高层推理）和 Grounder（视觉定位）两级。
- **Planner**：接收 8 方向全景图、粗略目标、历史、事实库、空间记忆、剩余预算，输出语义决策（ASK, MOVE, STOP）。
- **Grounder**：仅接收 Planner 选择的单方向视图和子目标，在 8×8 网格中选择一个单元作为局部航点。
- **执行器**：将网格单元反投影到世界坐标，通过射线检测处理碰撞，更新空间记忆。
- **成本感知交互**：Planner 在每次调用中观察剩余预算和问题类型代价，仅在预期不确定性降低大于成本时提问。Oracle 根据类型提供受限的自然语言答案（无度量坐标、无精确路径）。
- **空间记忆**：语言级记录，包括当前地点、目标事实、访问区域、未探索开口、路线提示、阻塞移动等。

## 6. 主要结论与发现

1. **成本感知交互有效**：TANDEM 在 Weighted SR@1.5 上达 21.4，远超所有基线（第二 GTA 仅 15.9），且使用较少问题获得更高成功。
2. **问题类型互补**：外观/区域问题降低早期目标不确定性→路线问题在歧义分支解决空间模糊→确认问题避免停在干扰物上。
3. **无交互严重退化**：TANDEM w/o QA 的 SR@1.5 降至 20.0，表明即使有完整描述和结构化记忆，也无法替代针对性交互。
4. **空间记忆关键**：移除空间记忆使 SR 下降 7.3 点。
5. **难度越高，交互价值越大**：困难集上，Type 3 提问使探索范围降低 47.5%。
6. **交互效率**：TANDEM 在困难集平均提 5.33 个问题，而朴素互动基线提 9.76 个问题但成功率更低，说明本方法提问更高效。

## 7. 优点

1. **问题类型分类基于数据驱动**：从五个人类语料库中挖掘线索并导出成本权重，可复现且可解释。
2. **加权成功率指标新颖**：首次将查询成本集成到导航评估中，能区分“高效消歧”与“过度依赖 oracle”。
3. **模块化零样本设计**：TANDEM 分离高层规划与低层执行，避免 MLLM 直接输出度量值，减少 3D 空间推理失败。
4. **交互协议可控**：oracle 回答受类型限制（无坐标、无精确路径），避免直接泄露答案，保证评估公平。
5. **诊断工具丰富**：基准包含难度分层、问题类型追踪、空间记忆记录，可细粒度分析失败原因。
6. **跨基准泛化**：在 CVDN/SOON/REVERIE 上也取得优于监督方法的成绩（如 REVERIE Test unseen SR=63.1 vs. 次优 57.7）。

## 8. 不足与局限

1. **仅限模拟环境**：所有实验在 Isaac Sim 中进行，假设理想化感知和定位。未在真实机器人上验证，噪声、漂移、深度缺失等现实问题未建模。
2. **MLLM 骨干依赖**：性能随骨干模型强度提升，但较弱的模型（如 4B 级别）下降明显，可能无法在边缘设备部署。
3. **交互延迟未考虑**：多次 MLLM 调用增加延迟，不适合需要实时响应的场景。论文未评估时间成本。
4. **问题类型可能不完整**：四种类型从语料库归纳，但可能遗漏特殊场景（如动态障碍物、多人指示）。
5. **oracle 设计理想化**：oracle 始终正确且准确，未考虑人类 oracle 的不确定性或错误。
