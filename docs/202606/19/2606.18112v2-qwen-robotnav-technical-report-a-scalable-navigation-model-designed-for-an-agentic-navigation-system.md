## 摘要
智能体导航系统需要一个基础导航模型，其观测策略可以在推理时由外部重新配置，因为指令跟随、物体搜索、目标跟踪和自动驾驶共享相同的感知-规划主干，但在消耗视觉流方面需要根本不同的策略。我们提出Qwen-RobotNav，这是一种基于Qwen-RobotNav构建的可扩展导航模型，通过一个带有两个互补维度的参数化接口来解决这一问题：多个任务模式用于选择导航行为，以及可控的观测参数（例如，令牌预算、每摄像机权重）用于控制视觉历史的编码方式。通过在训练时对所有参数进行随机化，Qwen-RobotNav对任何推理时的配置都具有鲁棒性，无需对Qwen-RobotNav主干进行任何架构修改。我们在1560万个样本上训练Qwen-RobotNav；与视觉-语言数据联合训练防止了在仅轨迹训练中观察到的退化到反应性动作序列映射器。参数化接口也使Qwen-RobotNav成为智能体系统的自然构建模块：对于长时域场景，上层规划器将目标分解为子任务，并在情节中动态切换Qwen-RobotNav的任务模式和上下文策略，通过重复调用同一模型来组合复杂行为。大量实验表明，Qwen-RobotNav在主要导航基准测试中达到了新的最先进结果。该模型从2B到8B参数表现出良好的扩展性，联合多任务训练发展出一个跨任务族转移的共享空间规划基底，并在不同环境中对真实世界机器人展示了强大的零样本泛化能力。

## Abstract
Agentic navigation systems require a base navigation model whose observation strategy can be externally reconfigured at inference time, because instruction following, object search, target tracking, and autonomous driving share the same perception-planning backbone yet demand fundamentally different strategies for consuming the visual stream. We present Qwen-RobotNav, a scalable navigation model built on Qwen-RobotNav that addresses it through a parameterised interface with two complementary dimensions: multiple task modes that select the navigation behaviour, and controllable observation parameters (e.g., token budget, per-camera weights) that govern how visual history is encoded. With training-time randomization over all parameters, Qwen-RobotNav is robust to any inference-time configuration requiring zero architectural modification to the Qwen-RobotNav backbone. We train Qwen-RobotNav on 15.6M samples; co-training with vision-language data prevents the collapse into reactive action-sequence mappers observed in trajectory-only training. The parameterised interface also makes Qwen-RobotNav a natural building block for agentic systems: for long-horizon scenarios, an upper-level planner decomposes goals into sub-tasks and dynamically switches Qwen-RobotNav's task mode and context strategy mid-episode, composing complex behaviours from repeated calls to the same model. Extensive experiments show that Qwen-RobotNav sets new state-of-the-art results across major navigation benchmarks. The model exhibits favourable scaling from 2B to 8B parameters, with joint multi-task training developing a shared spatial-planning substrate that transfers across task families, and demonstrates strong zero-shot generalisation to real-world robots across diverse environments.

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

**核心思想**：将多任务导航统一为“可重配置的航路点轨迹预测”问题。通过一个**参数化观测编码接口**，允许在推理时动态调整视觉上下文的编码方式（如保留多少历史、每帧的分辨率、各相机的权重），以适应不同任务的需求。通过**训练时随机化**所有这些参数，使模型学会在任何配置下工作，无需针对特定任务进行微调。

**关键技术细节**：

1.  **统一任务公式**：模型输入为文本指令 \( L \) 和多视角图像序列 \( I \)，输出为 \( K=8 \) 个航路点 \( W = \{(x_k, y_k, \theta_k)\}_{k=1}^{K} \)。

2.  **模型架构**：基于 Qwen3-VL，由三部分组成：
    -   **视觉编码器**：SigLIP-2 ViT，支持动态分辨率，便于后续的令牌分配。
    -   **语言主干**：Qwen3-VL 的 LLM，处理拼接后的视觉和语言令牌。
    -   **动作头**：一个轻量级的 4 层 MLP，将主干输出的隐藏状态映射为 24 维（8航路点 × 3自由度）的航路点坐标。

3.  **任务自适应观测编码**：这是核心创新，通过四个控制轴的参数接口实现：
    -   **视觉令牌预算 (B)**：控制用于描述所有视觉信息的令牌总数。
    -   **时间衰减 (γ)**：控制对最新帧的偏重程度，\( \gamma=0 \) 为均匀采样，\( \gamma \) 越大则越偏重最近帧。权重计算公式为：\( \omega_t = \exp(\gamma \cdot t / (T'-1)) \)。
    -   **相机权重 (w_c)**：为不同视角的相机（如前、后、左、右）分配不同的重要性。
    -   **帧采样模式 (m)**：决定是随机采样以覆盖全局历史，还是选择最新帧以获得高时效性。
    -   **算法流程**：根据配置选择一个子集帧 \( T' \)，计算时间权重和相机权重，形成联合权重矩阵 \( W[t, c] = \omega_t \cdot w_c \)，然后通过有约束的分配算法（先分配最小值 \( b_{min} \)，再按比例分配剩余预算）确定每张图片的令牌数，最后据此调整图片分辨率并输入模型。训练时 \( B, \gamma, w_c, b_{min}, b_{max} \) 等参数均在各自范围内随机采样。

4.  **外部化标识（Viewpoint and Temporal Identification）**：
    -   使用纯文本来标识帧的时间步和相机视角（例如，`Time step 1 Camera: Front <image>`）。

5.  **具身感知提示设计**：
    -   在系统提示中使用自然语言文本（如“Imagine you are a robot/car...”）来告诉模型其物理形态（机器人、汽车），从而激活其预训练世界中对应的知识（室内空间布局或交通规则）。

1.  **参数化观测策略有效**：Qwen-RobotNav 提出的任务自适应观测编码接口是成功的，通过一个统一的模型，可以在指令跟随、物体搜索、目标跟踪等多个任务上达到 SOTA 或极具竞争力的性能。
2.  **智能体系统的理想构建块**：该模型成功作为“可重配置执行器”嵌入智能体系统，在 EQA 任务上大幅超越现有方法（如 FAST-EQA），同时所需的导航步数减少了 77%。
3.  **联合训练至关重要**：与视觉语言数据联合训练能有效防止模型退化为“反应式映射器”，保留其开放世界感知和语言理解能力，这对于零样本泛化到新场景至关重要。
4.  **良好的扩展性**：模型性能从 4B 扩展到 8B 参数时持续提升，尤其在需要长期推理的任务（如 RxR 和 EQA）上表现更明显。
5.  **强大的零样本泛化**：在未见过的仿真环境和真实世界场景中均表现出强大的零样本泛化能力，证明了其统一设计的优势。

### 7. 优点

1.  **方法上的创新性**：将“观测上下文建模”提升为核心问题，并提出了一个优雅的、可外部控制的参数化接口。这重新定义了多功能机器人导航模型的构建方式，使其从“固定策略”转变为“可重配置原语”。
2.  **设计的简洁与优雅**：
    -   使用自然语言标签而非结构化的位置编码来传达时间和视角信息，零成本利用了预训练模型的语言能力。
    -   通过自然语言的系统提示来区分不同具身形态（机器人 vs 汽车），无需任何架构改动。
3.  **实验的广泛性与严谨性**：覆盖了所有主流导航基准，并进行了细致的消融分析和公平性对比。特别是对 HM3D 版本差异和观测模态差异的说明，体现了实验的严谨。
4.  **面向真实应用的系统设计**：将模型设计与上层智能体系统（规划器、记忆模块）集成作为核心目标，并展示了在实物机器人上的成功部署。这使得研究不仅停留在仿真层面，更具备实际应用价值。

### 8. 不足与局限

1.  **长期任务上的性能差距**：在具身问答上，虽然表现突出，但在某些子项（如 EXPRESS-Bench 的 Epath）上，系统的效率仍有提升空间。在长期自动驾驶仿真（AlpaSim）上的零样本表现不如专用模型，表明在具有严格安全约束的复杂动态环境中，模型仍有差距。
2.  **资源消耗与实时性**：8B 模型需要 H100 级别的算力进行训练。在推理端，尽管实现了 5 Hz 左右的推理速度（远程服务器），但对于某些对延迟极其敏感的任务（如高速目标跟踪或自动驾驶），这个速度仍可能是一个瓶颈。最近邻帧的处理虽然有优先级，但全局历史的重编码仍消耗资源。
3.  **观测编码策略的局限性**：论文提出的启发式令牌分配策略虽然有效，但作者自己也承认这并不是最优的。一个更“原则性”的、数据驱动的或可学习的分配算法（例如基于注意力机制）理论上可能带来更好的性能。
4.  **训练数据的依赖**：尽管使用了自动生成的数据，但大部分训练数据仍依赖于有昂贵重建成本的仿真环境（如 Habitat-Matterport 3D）。域迁移（sim-to-real）问题虽然通过图像增强和自动生成数据得到缓解，但并未完全解决。模型的成功很大程度依赖于精心设计和平衡的 1560 万训练样本。
