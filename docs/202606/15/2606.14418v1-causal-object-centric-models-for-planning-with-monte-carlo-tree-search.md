## 摘要
我们提出了COMET（因果对象中心高效树搜索模型），一种基于模型的强化学习算法，在插槽结构化的潜在空间中进行蒙特卡洛树搜索。COMET将冻结的无监督对象中心编码器与基于变换器的世界模型配对，在该模型中，通过一种新颖的动作-插槽融合机制将动作绑定到对象上，该机制用于插槽转移预测。策略和价值头使用对象-因果注意力，通过学习的每插槽相关性分数调节令牌交互，使决策集中在任务相关实体上。COMET为MuZero风格的潜在规划增加了明确的对象级归纳偏置。在来自Object-Centric Visual RL基准测试、ManiSkill、Robosuite和VizDoom的八个视觉和动态多样化的任务中，与对象中心和单一基线相比，COMET在训练早期阶段实现了更高的平均归一化分数。

## Abstract
We introduce COMET (Causal Object-centric Model for Efficient Tree search), a model-based reinforcement learning algorithm that performs Monte Carlo Tree Search in a slot-structured latent space. COMET pairs a frozen unsupervised object-centric encoder with a transformer-based world model, in which actions are bound to objects through a novel action-slot fusion mechanism that is used in slot transition prediction. Policy and value heads use object-causal attention, modulating token interactions by learned per-slot relevance scores so that decision-making concentrates on task-relevant entities. COMET adds an explicit object-level inductive bias to MuZero-style latent planning. Across eight visually and dynamically diverse tasks from the Object-Centric Visual RL benchmark, ManiSkill, Robosuite, and VizDoom, COMET achieves a higher mean normalized score during the early stages of training compared to object-centric and monolithic baselines.

- **动机**：人类能够将世界视为由离散对象组成，从而进行高效推理和规划。对象中心表示学习（如 Slot Attention）能从原始图像中无监督地分解出对象级表示，但现有对象中心模型基强化学习方法（如 COBRA、FOCUS、OC-STORM）存在交互建模不足、依赖标注掩码或缺乏因果推理等局限。
- **整体含义**：本文提出 COMET，旨在将对象级归纳偏置引入 MuZero 风格的潜在规划中，通过在槽结构潜空间中进行蒙特卡洛树搜索（MCTS），并利用因果注意力聚焦任务相关对象，从而提升样本效率和决策可解释性。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- COMET 将冻结的无监督对象中心编码器与基于 Transformer 的世界模型相结合，在槽（slot）结构化的潜在空间中执行 MCTS。
- 引入动作-槽融合（action-slot fusion）机制，将动作与每个对象槽绑定，用于槽转移预测。
- 策略和价值网络采用对象因果注意力（object-causal attention），通过学习的每槽相关性分数（causality score）调节令牌交互，使决策集中于任务相关实体。

### 2.2 关键技术细节
1. **槽提取器（Slots Extractor）**：
   - 将图像观测 $o_t$ 映射为一组对象槽 $\bar{s}_t = \{s^1_t, \dots, s^n_t\}$，n 为预设最大对象数（含背景）。
   - 为保持时间一致性，下一时间步的槽初始化使用当前时间步的槽表示。
   - 使用预训练的 SLATE、DINOSAUR 或 Slot Contrast 作为编码器，并在强化学习期间冻结。

2. **对象中心令牌处理（Object-Centric Token Processing）**：
   - 基于 UniZero 的 Transformer 架构（nanoGPT）改造。
   - 状态由 n 个槽 $\bar{s}_t$ 表示，经过 Transformer 骨干得到潜在表示 $\{h^1_t, \dots, h^n_t\}$。
   - 对于转移预测：将动作嵌入 $a_t$ 与每个槽 $s^i_t$ 拼接，通过共享 MLP 投影器得到槽条件动作嵌入 $\bar{a}_t = \{a^1_t, \dots, a^n_t\}$，再送入 Transformer 得到 $\bar{z}_t = \{z^1_t, \dots, z^n_t\}$，最后通过共享观测头预测下一时间步的槽 $\hat{\bar{s}}_{t+1}$。
   - 奖励预测：将 $\bar{z}_t$ 求和后经奖励头 MLP 得到。
   - 注意力掩码设计：每个槽可以关注同块内所有槽、之前所有块以及其位置对应的动作嵌入；动作嵌入可关注自身及当前和之前的槽。

3. **对象因果注意力（Object-Causal Attention）**：
   - 策略和价值网络使用修改的 Transformer 注意力机制。
   - 引入一个可学习的目标令牌（target token）用于策略或价值预测。
   - 定义因果图 $G$：目标受因果对象影响，因果对象间互影响，非因果对象只影响自身。
   - 为每个槽学习一个因果分数 $\alpha^i_t \in [0,1]$，表示该对象对预测的相关性。
   - 注意力计算时，用 $W_t G W_t^\top$ 调制缩放点积注意力，其中 $W_t$ 是由目标令牌和所有 $\alpha^i_t$ 构成的矩阵，使得相关对象获得更高注意力权重。

4. **策略与世界模型学习**：
   - 遵循 MuZero 风格：表示函数 $h_\theta$、动态函数 $g_\theta$、预测函数 $f_\theta$。
   - 在潜空间中进行 MCTS，使用访问计数生成改进策略目标 $p_t$。
   - 联合优化损失函数 $L_{\text{COMET}}$，包含：
     - 下一槽状态预测损失（MSE）
     - 奖励预测损失（交叉熵）
     - 策略预测损失（交叉熵）
     - 价值预测损失（交叉熵）
     - 各损失项由系数 $\beta_s, \beta_r, \beta_p, \beta_v$ 平衡。
   - 奖励和价值预测作为对数变换空间中的离散回归问题，使用软目标交叉熵优化。

- **动作-槽融合机制**：将动作与每个槽独立拼接并通过共享 MLP 投影，避免单动作嵌入瓶颈，使转移预测能保留对象级信息。
- **对象因果注意力**：显式建模对象与目标令牌之间的因果相关性，提升决策聚焦能力，且具备可解释性（可视化因果分数）。
- **统一 Transformer 框架**：将状态、动作、策略/价值预测整合到一个 Transformer 骨干中，简化架构，并与 MuZero 风格 MCTS 兼容。
- **跨基准评估**：涵盖 Object-Centric RL benchmark、ManiSkill、Robosuite、VizDoom，验证了在多种视觉和动力学条件下的泛化能力。

## 8. 不足与局限
- **依赖高级编码器**：COMET 性能高度依赖冻结编码器的质量。目前无监督对象中心表示在复杂场景（如 Block Lifting 中机械臂与物体界限模糊）下仍不可靠，导致槽分配不稳定。
- **缺乏消融研究**：未对因果注意力、动作-槽融合等组件进行消融实验，难以量化各模块贡献。
- **计算扩展性问题**：Transformer 的二次自注意力复杂度随槽数量增长，限制了在多对象场景中的可扩展性（论文提到这一点作为未来方向）。
- **因果注意力的局限性**：在 Defend the Line 任务中，几乎所有对象都相关，注意力无法有效区分；在 Object Interaction 中，目标通过间接因果链条达成，因果分数可能误导，导致性能与基线相当。
- **基线比较不全面**：未与更多最新对象中心 MBRL 方法（如 STICA、OC-STORM 等）比较；SOLD 仅用于连续任务，且编码器不同。
- **实验环境局限**：所有仿真环境，未在真实机器人上验证。任务步数少（最多 125 步），无法检验长期规划能力。
- **资源披露不完整**：仅提供了 COMET 的训练资源，未提供基线方法的资源消耗，不利于公平比较计算效率。
