## 摘要
视觉语言模型（VLM）已成为机器人领域中一种强大的工具，能够结合语言上下文实现环境感知，为开放词汇任务（如ObjectNav）提供了催化剂。然而，其计算开销通常使其局限于云端执行，阻碍了在资源受限机器人上的本地低延迟推理部署。为解决这一挑战，我们提出了一种蒸馏策略，将大型前沿模型的复杂空间-语义推理能力迁移至轻量级40亿参数本地VLM，用于嵌入式GPU设备（如Jetson Orin）上的边缘执行。我们首先使用Claude Sonnet 4.6建立了一个基于场景图（SG）的最先进（SotA）流水线，在HM3D OVON基准上实现了39.7%的成功率（SR）。随后，我们证明仅使用500条前沿推理轨迹微调Qwen3.5-4B即可有效赋予导航能力，达到34.5%的SR，缩小了与大型云模型性能的差距。最后，我们引入带有令牌生成（TG）正则化的E-RLVR，以压缩物理部署时的输出序列长度，同时将智能体锚定在其任务中。这一下游优化使TG开销降低72.1%，延迟降低71.8%。结合量化，该联合策略在未显著牺牲性能的情况下，累计降低了82.8%的整体推理延迟，为移动机器人上的本地低延迟VLM执行提供了一种可行范式。

## Abstract
Vision Language Models (VLMs) have emerged in the robotic domain as a powerful tool that enables environmental perception with language context, serving as a catalyst for open-vocabulary tasks like ObjectNav. Yet, their computational footprint typically confines them to cloud execution, hindering low-latency inference with local deployment on resource-constrained robots. To address this challenge, we present a distillation strategy that transfers complex spatial-semantic reasoning from large frontier models into a lightweight, 4B-parameter local VLM for edge execution on embedded GPU devices (e.g., Jetson Orin). We first establish a State of the Art (SotA), Scene Graph (SG)-based pipeline using Claude Sonnet 4.6, achieving a 39.7% Success Rate (SR) on the HM3D OVON benchmark. We then demonstrate that fine-tuning Qwen3.5-4B on just 500 frontier reasoning traces effectively enables navigation capabilities, yielding a SR of 34.5%, narrowing the gap to the performance of large cloud models. Finally, we introduce E-RLVR with Token Generation (TG) regularization to compress output sequence lengths for physical deployment while grounding the agent in its task. This downstream optimization reduces TG overhead by 72.1% and latency by 71.8%. Combined with quantization, this joint strategy yields a cumulative 82.8% reduction in overall inference latency without significantly sacrificing performance, presenting a viable paradigm for local, low-latency VLM execution on mobile robots.

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：采用“先蒸馏、后优化”的两阶段流水线。首先，将大型云模型的场景理解与推理能力蒸馏到一个轻量级的学生模型（4B参数）；然后，通过具身强化学习（Embodied RL）对该模型进行微调，使其输出更简洁、更高效，以适应边缘设备的计算和延迟限制。
- **关键技术细节**：
    1.  **模块化系统架构**：
        - **高层决策（VLM）**：系统不直接预测低级动作，而是让VLM基于**实时构建的3D场景图（Scene Graph, SG）** 进行高层推理，输出语义动作（如 `navigate(object id)`, `explore(room id)`）。
        - **低层控制**：高级决策（目标点）交由一个独立的、轻量级的PointGoal导航策略执行，负责避障和路径规划。
        - **场景表征**：将SG信息以**文本（列出物体、房间）** 和**增强图像（在RGB图像上叠加物体ID）** 两种形式作为VLM的输入，使模型能进行显式空间推理。
    2.  **蒸馏数据生成（SFT阶段）**：
        - **过程**：使用**Claude/GPT/Gemini**等大型云模型在模拟环境中执行任务，仅记录其成功轨迹中的“图像-SG状态-推理轨迹”作为**监督微调（SFT）** 数据集。
        - **优势**：无需复杂特征对齐，数据量极小（仅约 **500条轨迹/每个教师模型**），操作简单且模型无关。
    3.  **具身强化学习与令牌生成优化（E-RLVR阶段）**：
        - **目标**：在SFT后，模型输出冗余，导致高延迟。E-RLVR旨在压缩输出长度，同时保持任务性能。
        - **过程**：使用**基于群体的相对策略优化（GRPO）在闭环模拟环境中训练**。在每个决策点，采样多个（N=4）候选动作，让模型“边想边做”。
        - **奖励函数**：采用多目标累积奖励，包含：
            - `R_done`：导航成功与否的奖励。
            - `R_nav`：朝向目标前进的正奖励。
            - `R_exp`：探索新区域的奖励。
            - `R_brevity`：**关键创新**，惩罚过长的输出，奖励更简洁的生成，强制模型内化“节俭”策略。

### 6. 论文的主要结论与发现

- **蒸馏有效性**：仅用约500条教师推理轨迹进行SFT，就能将4B小模型的SR提升至34.5%，性能接近云模型的39.7%（使用Claude作教师）。
- **教师模型影响**：学生模型的性能并不完全与教师模型能力正相关。最强的教师模型（Gemini）蒸馏出的学生性能（41%SR）不如稍弱的教师（GPT，43%SR），Claude蒸馏的学生综合表现最佳（47%SR）。
- **E-RLVR的关键作用**：E-RLVR能有效压缩模型输出长度（由535.2降至149.36 tokens），降低71.8%的端到端推理延迟，同时**不降低**（甚至略微提升）导航性能（49% SR）。
- **量化增效**：结合量化（IQ4-XS），可进一步将TG吞吐率提升一倍以上。两者结合实现了累计82.8%的整体延迟降低。
- **核心发现**：**SFT负责学习能力，E-RLVR负责优化部署效率**。两者是互补且缺一不可的，单独使用无法同时兼顾高性能和低延迟。
