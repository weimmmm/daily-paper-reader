## 摘要
基于大型预训练模型的视觉导航策略至今遵循一个通用配方：专用的视觉编码器、定制的动作头以及在数千小时跨实体数据集上的训练。我们质疑这个配方是否必要。本文介绍GemNav，一种视觉机器人导航策略，它通过仅在语言塔上使用低秩自适应（LoRA）来适配冻结的多模态大语言模型（MLLM）以实现短中期航点导航，无需辅助视觉编码器和连续回归头。航点与分类导航信号共享由语言模型头生成的单一离散标记词汇表，而软解码的辅助损失恢复了纯交叉熵训练所丢弃的度量结构。在单一的8.7小时开放语料库上——比竞争训练集小约三个数量级——该策略零样本迁移到四个物理上不同的未见环境，并在涵盖开放停车场、障碍停车场、室外长化工园区和室内仓库的20次真实世界试验中，在距离目标0.25-0.42米内停止。基于短图像历史的条件化改进了离线指标但对机器人无益，表明一旦预训练视觉特征就位，时间上下文所能增加的收益存在上限。这些结果表明，冻结MLLM的离散标记适应可以为基础模型机器人导航提供一种数据高效且可部署的替代方案。

## Abstract
Visual navigation policies built on large pretrained models have so far followed a common recipe: a dedicated visual encoder, a bespoke action head, and training on thousands of hours of cross-embodiment datasets. We ask whether this recipe is necessary. In this paper, we introduce GemNav, a visual robot navigation policy that adapts a frozen Multimodal Large Language Model (MLLM) for short-to-medium horizon waypoint navigation using Low-Rank Adaptation (LoRA) on the language tower alone, with no auxiliary visual encoder and no continuous regression head. Waypoints and categorical navigation signals share a single discrete token vocabulary generated by the language-model head, and a soft-decoded auxiliary loss recovers the metric structure that pure cross-entropy training discards. On a single 8.7-hour open corpus, roughly three orders of magnitude smaller than competing training sets, the policy transfers zero-shot to four physically distinct unseen environments and stops within 0.25-0.42m of the goal across 20 real-world trials covering an open carpark, an obstacle carpark, a long outdoor chemical yard, and an indoor warehouse. Conditioning on short image histories improves offline metrics but yields no robot benefit, pointing to a ceiling on what temporal context adds once pretrained vision features are in place. These results indicate that discrete-token adaptation of frozen MLLMs can provide a data-efficient, deployable alternative for foundation model robot navigation.

## 1. 核心问题与整体含义
**研究动机与背景：**
当前视觉导航策略（如 OmniVLA、ViNT、NoMaD）遵循一个通用范式：专用视觉编码器 + 定制连续动作回归头 + 数千小时跨实体数据集训练。本文质疑该范式的必要性，探索**能否仅利用冻结的、已预训练的多模态大语言模型（MLLM）实现数据高效的机器人导航**，将导航问题转化为离散 token 序列预测，从而绕过视觉编码器瓶颈和大量数据需求。

## 2. 方法论：核心思想与关键技术
**核心思想：**
- 将航点（连续坐标）和导航信号（到达/不可到达）统一映射为 MLLM 语言头的离散 token 输出，无需额外回归头或视觉编码器。
- 仅对 MLLM 的语言塔应用 LoRA（r=32, α=16），视觉塔完全冻结。

**关键技术细节：**
- **离散 token 化**：每个航点的 x/y 轴使用 64 个均匀值箱（范围 ±15 m，每箱 0.469 m），加上特殊 token（`<bot_traj>`、`<eot_traj>`、`<goal_reached>`、`<goal_unreachable>`、`<goal_coord>`）。输出为一个 18 令牌序列（8 个航点 + 边界标记）或停止序列。
- **软解码辅助损失**：在交叉熵基础上，对值箱 token 的 softmax 分布加权平均得到连续预测，并计算 MSE 损失（λ=0.1），恢复度量结构（验证 ADE 降低 23%）。
- **提示构造**：包含任务描述、当前图像帧、可选历史帧、目标图像或坐标（`<goal_coord>` + 两个值箱 token），最终触发轨迹预测。
- **训练**：bf16、AdamW（lr=1e-4）、梯度检查点、batch size=1 per device、40k 步/阶段、四阶段热启动链（A: 新鲜 LoRA → B: 温启动 → C1: 联合模态专化 → C2: 全模态泛化）。

## 3. 实验设计
**数据集与场景：**
- **训练集**：SCAND（约 8.7 小时，Spot/Jackal 遥操作数据，仅使用图像和位姿）。
- **OOD 验证集**：TokenWalker（83 分钟，同一平台拍摄的室外场景）。
- **部署环境**：4 个真实未见场景：开放停车场（~10-12 m）、障碍停车场（~18 m）、室外化工园区（~30-34 m）、室内仓库（~32 m）。每个场景 5 次独立试验（n=5）。

**基准方法：**

- 离线指标：平均位移误差（ADE）和终点位移误差（FDE）。

**模态设置：**

- **消融实验**：覆盖解码模式、历史深度（h=0,2,4）、目标采样方式（距离 vs 帧对齐）、LoRA 秩、裁剪模式、辅助损失、数据集混合等。实验设计较全面，但由于硬件限制，部分 OOD 结果存在验证集构造不匹配的问题（附录 E.3 已指出）。

## 6. 主要结论与发现
- **冻结 MLLM + 离散 token 可数据高效部署**：仅 8.7 小时数据训练，零样本迁移到 4 个未见环境，pose 模式成功率 100%，停止距离 0.25-0.42 m，接近人类遥操作水平。
- **历史深度对闭环部署无益**：增加视觉历史（h=2,4）改善离线 ADE（从 1.45m 降至 1.15m），但实际机器人性能下降（h=4 模态成功率降至 4/5，位移增大），存在时间滞后效应。
- **软解码辅助损失关键**：消除纯 CE 的度量无感问题，ADE 降低 23%。
- **距离基目标采样优于帧对齐**：帧对齐采样虽然离线 ADE 更低，但实际部署完全失败（立即触发 stop），而距离基采样成功跨所有场景。
- **动作表示设计比进一步扩展视觉更重要**：通过重构动作表示（离散 token + 辅助损失）即可实现可部署性能，无需视觉缩放。

## 7. 优点
- **架构极简**：冻结 MLLM + LoRA，无需额外视觉编码器或专用动作头，训练参数极少。
- **数据高效**：仅 8.7h 单数据集训练，比 SOTA 方法（OmniVLA 9500h）小三个数量级。
- **统一 token 接口**：连续航点、停止信号、不可达决策均以离散 token 输出，模型输出与 LM 原生生成接口一致。
- **软解码辅助损失**：巧妙恢复度量结构而不增加参数，验证了其有效性。
- **真实机器人验证充分**：涵盖室内外、障碍、长距离等多样场景，且与人类操作对比。

## 8. 不足与局限
- **单平台训练与部署**：仅在波士顿动力 Spot 上验证，未测试跨实体迁移（如轮式机器人、无人机）。
- **图像目标停止不可靠**：纯图像目标（ego 模式）停止成功率仅 1/5，依赖坐标目标，限制了部署灵活性。
- **无法处理原地旋转**：输出仅为 2D 平移航点，不支持旋转动作。
- **时间滞后效应**：历史帧引入导致闭环响应滞后，可能引发超调问题（尽管离线指标提升）。
- **OOD 验证集构造问题**：部分 OOD 评估使用早期验证集（位置滞后），可能影响结果可比性（附录 E.3 已注明）。
- **未公开完整代码与权重**：论文承诺发布 TokenWalker 数据集部分内容，但未提及完全开源模型权重。
