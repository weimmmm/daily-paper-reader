## 摘要
跨具身迁移在视觉-语言-动作（VLA）模型中仍然具有挑战性，因为不同机器人平台的低级状态和动作空间存在根本差异。我们观察到，操作背后的高级认知过程，包括场景感知、物体识别、任务规划和子任务分解，在很大程度上是跨具身共享的。基于这一观察，我们提出了ZR-0，一个26亿参数的端到端VLA模型，该模型使用密集的具身思维链（ECoT）监督来对齐视觉-语言模型（VLM）中的跨具身表示。ZR-0采用双流架构：一个预训练的VLM（系统2）在训练期间生成结构化的ECoT推理，而一个基于扩散Transformer的动作专家（系统1）通过流匹配生成连续的动作块。这两个组件通过交叉注意力耦合，其中注意力掩码限制动作专家仅处理输入提示特征，使得推理时可以完全跳过ECoT生成而没有任何性能损失。ZR-0在ProcCorpus-60M上进行预训练，这是一个包含约6000万帧（约1000小时）来自超过40万条轨迹的大规模数据集，密集的ECoT注释覆盖了所有帧的96.8%。我们在三个模拟基准上评估ZR-0，涵盖单臂（LIBERO）、双臂（RoboTwin 2.0）和人形（RoboCasa GR-1 Tabletop）具身，以及在xArm平台上的真实世界实验，展示了在所有设置下的强大性能。代码和模型检查点可在https://github.com/RUCKBReasoning/ZR-0获取。

## Abstract
Cross-embodiment transfer in vision-language-action (VLA) models remains challenging because low-level state and action spaces differ fundamentally across robot platforms. We observe that the high-level cognitive process underlying manipulation, including scene perception, object identification, task planning, and sub-task decomposition, is largely shared across embodiments. Based on this observation, we present ZR-0, a 2.6 billion parameter end-to-end VLA model that uses dense Embodied Chain-of-Thought (ECoT) supervision to align cross-embodiment representations within the vision-language model (VLM). ZR-0 adopts a dual-stream architecture: a pre-trained VLM (System 2) generates structured ECoT reasoning during training, while a Diffusion Transformer-based action expert (System 1) produces continuous action chunks via flow matching. The two components are coupled through cross-attention, with an attention mask that restricts the action expert to input prompt features only, enabling ECoT generation to be entirely skipped at inference without any performance loss. ZR-0 is pre-trained on ProcCorpus-60M, a large-scale dataset comprising approximately 60 million frames (approximately 1,000 hours) from over 400K trajectories, with dense ECoT annotations covering 96.8% of all frames. We evaluate ZR-0 on three simulation benchmarks spanning single-arm (LIBERO), bimanual (RoboTwin 2.0), and humanoid (RoboCasa GR-1 Tabletop) embodiments, as well as real-world experiments on the xArm platform, demonstrating strong performance across all settings. Code and model checkpoints are available at https://github.com/RUCKBReasoning/ZR-0.

### **训练基于密集具身思维链的视觉-语言-动作模型 (ZR-0) 论文分析总结**

#### **1. 论文的核心问题与整体含义**

- **核心问题**：视觉-语言-动作 (VLA) 模型在进行**跨实体迁移**时面临巨大挑战。不同机器人（如单臂、双臂、人形）的低级状态空间（如关节角度）和动作空间（如末端执行器位姿）在物理意义上存在根本性差异，导致直接共享模型难以有效迁移能力。
- **研究动机**：观察到尽管低级控制信息是实体特定的，但**高级认知过程**（如场景感知、物体识别、任务规划、子任务分解）在不同实体间是高度共享的。因此，如果能利用高级认知过程作为“桥梁”来对齐VLM的表示，就可能实现有效的跨实体学习。
- **整体含义**：本文提出了一种名为**ZR-0**的VLA模型，其核心思想是利用**密集的具身思维链 (ECoT) 监督**来训练VLM，使其学习到与具体机械结构无关的、可供迁移的高级认知表示，从而显著提升模型的跨实体泛化能力。

#### **2. 论文提出的方法论：核心思想、关键技术细节**

- **核心思想**：采用“**系统1/系统2**”的双流架构。系统2（VLM）学习高级的、实体无关的认知推理（ECoT），系统1（动作专家）将这种高级表示解码为特定实体的低级连续动作，并通过**交叉注意力**机制解耦二者，使得推理时可跳过ECoT生成，降低延迟。

- **关键技术与架构**：
    1.  **双流架构 (Dual-Stream Architecture)**:
        *   **System 2 (VLM)**: 基于预训练VLM（Qwen3-VL-2B-Instruct）。输入：任务指令和图像。输出：特征表示$f_t$。训练目标：生成结构化的ECoT推理序列$r_t$。
        *   **System 1 (Action Expert)**: 基于Diffusion Transformer (DiT)。输入：VLM特征$f_t$和机器人状态$s_t$。输出：通过流匹配（Flow Matching）生成连续的动作块 $A_t$。
    2.  **交叉注意力耦合 (Cross-Attention Coupling)**:
        *   VLM的输出特征 $f_t$ 作为Key/Value，传给动作专家的交叉注意力层。
        *   **关键创新点：注意力掩码 (Attention Mask)**：动作专家**只允许**关注VLM中对应**输入提示**（任务指令和图像）的特征，而**忽略**ECoT生成的特征。这使得训练后，推理时只需一次VLM前向传播生成输入提示特征即可，无需昂贵的ECoT自回归生成。
    3.  **密集具身思维链 (Dense ECoT Supervision)**:
        *   每个训练帧都包含一个结构化的ECoT，由6个组件构成：
            *   **场景描述 (Scene Description)**: 提升物体识别能力。
            *   **进度评估 (Progress Assessment)**: 训练任务进度感知。
            *   **未来计划 (Future Plan)**: 训练长期时序推理与规划。
            *   **待办动作 (To-Do Actions)**: 将未来计划分解为原子子任务，是**跨实体对齐的关键**。
            *   **目标物体 (Target Objects)**: 边界框标注，提供视觉定位监督。
            *   **离散动作 (Discrete Actions)**: FAST分词器产生的离散动作令牌，连接高级推理与低级控制。
    4.  **训练目标 (Training Objective)**:
        *   **ECoT生成**: 标准的**下一个词预测损失** $L_{ntp}$，仅更新VLM参数。
        *   **连续动作生成**: **流匹配去噪损失** $L_{fm}$，更新整个模型（VLM和动作专家）。
        *   **总损失**: $L = L_{ntp} + \alpha L_{fm}$，其中$\alpha$是平衡权重。
    5.  **推理阶段**:
        *   **完全跳过ECoT生成**。直接从高斯噪声初始化动作块，然后通过流匹配的欧拉积分迭代去噪，生成最终动作。推理延迟约90ms，适合实时部署。
